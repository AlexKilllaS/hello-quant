# 低波动小市值 QMT 策略说明与优化指南

本文档对应策略文件：`quantcode/低波动小市值_qmt.py`。

这不是一份单纯的代码注释，而是一份学习型策略说明文档。它有两个目标：

1. 清晰描述当前 `低波动小市值_qmt.py` 到底在做什么。
2. 把你在 `docs` 里学到的量化选股、择时、调仓和回测知识，转化成后续优化这个策略时能直接使用的检查清单和实验路径。

重要边界：本文档解释当前策略如何工作，并给出优化实验方向。策略代码的量化逻辑仍以 `quantcode/低波动小市值_qmt.py` 为准。

## 0. 先建立全局地图

当前策略可以拆成 4 个学习模块：

| 学习模块 | docs 对应章节 | 当前策略中的位置 | 你要学会问的问题 |
| --- | --- | --- | --- |
| 量化选股 | `docs/ch04_量化选股策略` | `get_stock_list()`、`calc_barra_factors()`、`calc_composite_score()` | 为什么这些股票被选中，因子真的有用吗 |
| 量化择时 | `docs/ch05_量化择时策略` | `weekly_adjustment()`、`check_stop_loss()`、`trade_afternoon()` | 什么时候买卖，是否需要市场过滤 |
| 量化调仓 | `docs/ch06_量化调仓策略` | `buy_security()`、`order_target_value_()`、`g.stock_num` | 每只股票买多少，持仓是否太集中 |
| 量化回测 | `docs/ch07_量化回测` | QMT 回测结果和后续分析表 | 优化后是真的更好，还是只在某段行情碰巧更好 |

一句话概括当前策略：

```text
在中小盘股票池里，先做可交易性和风险过滤，
再用低波动、低换手相关因子排序，
每周买入综合得分最低的前 N 只股票。
```

## 1. 策略结构

当前 QMT 策略入口：

| 函数 | 作用 |
| --- | --- |
| `init(ContextInfo)` | 初始化股票池、账户、参数和全局状态 |
| `handlebar(ContextInfo)` | 每根新日线 bar 执行一次策略主流程 |

`handlebar()` 的执行顺序：

```text
更新交易日状态
-> 数据不足时跳过
-> prepare_stock_list()
-> weekly_adjustment()
-> check_stop_loss()
-> trade_afternoon()
-> 更新本地收益率变量
```

这里要注意：QMT 回测没有直接使用 JoinQuant 的 `run_daily` 定时任务，而是在 `handlebar()` 中通过“第几个交易日”和当前 bar 来模拟调度。

## 2. 当前策略属于哪类策略

从 docs 的角度看，它是一个“多因子截面选股 + 固定频率调仓 + 简单风控”的策略。

它不是纯择时策略。也就是说，它的核心收益来源不是判断大盘涨跌后决定满仓或空仓，而是每周在股票池内部重新排序，选择当前更符合因子条件的股票。

当前策略主要暴露在这些风险上：

| 风险暴露 | 来源 | 为什么要关注 |
| --- | --- | --- |
| 小市值风险 | `5 亿 <= 总市值 <= 50 亿` | 回测收益可能主要来自小市值 Beta |
| 流动性风险 | 小市值股票 + 低换手因子 | 实盘滑点和冲击成本可能放大 |
| 集中持仓风险 | 默认只持有 10 只 | 单只股票黑天鹅会明显影响净值 |
| 风格失效风险 | 偏低波动、低换手 | 强趋势牛市里可能跑输高弹性股票 |
| 数据字段风险 | 市值、涨跌停、停牌字段都依赖 QMT 本地数据 | 字段单位错了，排序就会错 |

## 3. 选股：把 ch04 多因子选股映射到代码

### 3.1 ch04 的学习框架

`docs/ch04_量化选股策略` 里，多因子选股大致是这条链路：

```text
确定目标和约束
-> 选择因子
-> 处理异常值
-> 标准化
-> 确定因子权重
-> 合成综合评分
-> 选出股票组合
-> 回测和调整
```

当前策略正好可以按这个框架拆开看。

| ch04 步骤 | 当前策略怎么做 | 对应代码 |
| --- | --- | --- |
| 目标和约束 | 只在指定股票池里选小市值、可交易股票 | `get_stock_list()` |
| 选择因子 | 使用低波动和低换手相关因子 | `calc_barra_factors()` |
| 异常值处理 | 对因子做 MAD 去极值 | `_mad_winsorize()` |
| 标准化 | 对因子做 Z-Score | `_zscore()` |
| 因子权重 | 7 个因子当前等权 | `g.factor_weights` |
| 综合评分 | 加权得到 `composite_score` | `calc_composite_score()` |
| 股票筛选 | 得分越低越优，取前 N 只 | `weekly_adjustment()` |

### 3.2 股票池

当前股票池参数：

| 参数 | 当前值 | 含义 |
| --- | --- | --- |
| `g.benchmark` | `000300.SH` | HSIGMA 使用的市场收益参考，也可作为回测基准之一 |
| `g.index_code` | `399101.SZ` | 初始股票池，代码里映射为 QMT 本地板块名 |

代码通过 `INDEX_SECTOR_NAME_MAP` 把指数代码映射到 QMT 本地板块名。你在 QMT 里跑回测前，首先要看日志里的股票池数量是否合理。如果股票池为空，后面的选股、调仓、回测指标都没有意义。

### 3.3 基础过滤

`get_stock_list()` 中的基础过滤顺序：

| 过滤器 | 当前逻辑 | 学习含义 |
| --- | --- | --- |
| `filter_listed_before` | 上市满约一年 | 避免新股数据不足或炒作期 |
| `filter_kcbj_stock` | 排除 4、8、688、300 开头股票 | 控制交易制度和风格差异 |
| `filter_st_stock` | 排除 ST、退市风险相关股票 | 基础风险过滤 |
| `filter_paused_stock` | 排除停牌或无有效量价数据股票 | 保证可交易 |
| `filter_limitup_stock` | 排除涨停且未持有股票 | 避免买不进去 |
| `filter_limitdown_stock` | 排除跌停且未持有股票 | 避免流动性风险 |
| `filter_highprice_stock` | 排除价格高于 `g.up_price` 的未持有股票 | 控制高价股和资金利用效率 |

新手优化时不要一开始就把这些过滤全部关掉。更好的方式是做“单项消融实验”：

```text
只关闭一个过滤器
-> 其他条件不变
-> 观察收益、回撤、换手、持仓数量变化
```

### 3.4 流动性过滤

当前参数：

| 参数 | 当前值 | 含义 |
| --- | --- | --- |
| `g.liquidity_days` | `20` | 成交额观察窗口 |
| `g.min_avg_amount` | `5e7` | 近 20 日平均成交额至少 5000 万元 |

代码逻辑：

```text
近 20 日平均成交额 >= 5000 万元
```

这一步很重要。小市值策略回测很容易好看，但实盘可能因为成交额太小、滑点太大而失真。

建议实验：

| 实验 | 观察重点 |
| --- | --- |
| 3000 万、5000 万、8000 万、1 亿 | 流动性门槛越高，收益是否下降，回撤是否改善 |
| 10 日、20 日、60 日窗口 | 短窗口更敏感，长窗口更稳定 |
| 分资金规模回测 | 100 万、500 万、1000 万资金下是否仍能成交 |

### 3.5 动量过滤

当前参数：

| 参数 | 当前值 | 含义 |
| --- | --- | --- |
| `g.momentum_days` | `30` | 动量观察窗口 |
| `g.filter_threshold1` | `-0.5` | 30 日跌幅下限 |
| `g.filter_threshold2` | `0.5` | 30 日涨幅上限 |

当前逻辑：

```text
-50% < 近 30 日涨跌幅 < +50%
```

这个过滤很宽，主要是排除极端暴跌和极端暴涨，不是强择时信号。它更像风险过滤。

建议实验：

| 参数组合 | 想验证什么 |
| --- | --- |
| `[-0.5, 0.5]` | 当前基准 |
| `[-0.3, 0.3]` | 排除更极端股票后，回撤是否下降 |
| `[-0.2, 0.5]` | 更严格规避暴跌股 |
| 关闭动量过滤 | 判断这个过滤到底有没有贡献 |

### 3.6 市值过滤

当前参数：

| 参数 | 当前值 |
| --- | --- |
| `g.min_market_cap` | `5` |
| `g.max_market_cap` | `50` |

当前逻辑：

```text
5 亿 <= 总市值 <= 50 亿
```

市值主要通过：

```text
close * TotalVolume / 1e8
```

流通市值主要通过：

```text
close * FloatVolume / 1e8
```

如果 QMT 合约详情字段缺失，代码会尝试使用日线行情中的市值候选字段兜底。这里一定要检查单位，因为市值单位错了会直接影响股票排序。

建议实验：

| 市值区间 | 目的 |
| --- | --- |
| 5 亿到 30 亿 | 更强小市值暴露 |
| 5 亿到 50 亿 | 当前基准 |
| 10 亿到 80 亿 | 降低微盘风险 |
| 20 亿到 100 亿 | 检查是否脱离微盘后仍有 Alpha |

## 4. 因子：当前策略怎么打分

当前使用 7 个 Barra 量价类因子：

| 因子 | 类型 | 当前理解 | 排序方向 |
| --- | --- | --- | --- |
| `HSIGMA` | 波动率 | CAPM 回归残差波动率 | 越低越好 |
| `DASTD` | 波动率 | 指数加权日收益标准差 | 越低越好 |
| `CMRA` | 波动率 | 过去 12 个月累计收益区间 | 越低越好 |
| `STOM` | 换手 | 月度成交量/市值近似 | 越低越好 |
| `STOQ` | 换手 | 季度换手聚合 | 越低越好 |
| `STOA` | 换手 | 年度换手聚合 | 越低越好 |
| `ATVR` | 换手 | 指数加权换手率 | 越低越好 |

合成方式：

```text
原始因子
-> MAD 去极值
-> Z-Score 标准化
-> 按 g.factor_weights 加权
-> 得到 composite_score
-> composite_score 越低越优
```

当前权重：

```python
g.factor_weights = {name: 1 / len(FACTOR_NAMES) for name in FACTOR_NAMES}
```

这等价于 7 个因子等权。

### 4.1 因子优化不要直接上复杂模型

按照 ch04 的思路，优化因子前先做三件事：

1. 做单因子测试：每次只用一个因子排序，看它是否稳定贡献收益。
2. 做因子组测试：低波动组和低换手组分开测试。
3. 做因子相关性检查：如果两个因子高度相似，等权可能只是重复押注同一种风格。

建议实验：

| 实验 | 目的 |
| --- | --- |
| 只用 `HSIGMA/DASTD/CMRA` | 测试低波动因子族 |
| 只用 `STOM/STOQ/STOA/ATVR` | 测试低换手因子族 |
| 去掉单个因子 | 找出拖后腿因子 |
| 波动率组 70%、换手组 30% | 偏防御 |
| 波动率组 30%、换手组 70% | 偏低换手/低拥挤 |
| 等权 vs 手动权重 | 判断复杂权重是否真的优于简单等权 |

### 4.2 市值中性化是后续进阶点

ch04 里提到市值中性化和行业中性化。当前策略没有做中性化，而且还主动筛选了小市值股票，所以它很可能同时包含：

```text
小市值 Beta + 低波动/低换手 Alpha
```

你后续要重点验证：收益到底来自因子本身，还是主要来自小市值暴露。

可做实验：

| 实验 | 判断 |
| --- | --- |
| 和中证 1000/中证 2000 比较 | 小盘指数能否解释大部分收益 |
| 放宽市值到 20-100 亿 | 因子脱离微盘后是否仍有效 |
| 分市值组回测 | 小市值、中市值里因子表现是否一致 |
| 行业分布统计 | 是否集中在少数行业 |

## 5. 择时：当前策略有什么、缺什么

### 5.1 当前策略不是典型择时策略

ch05 讲的择时，重点是用客观技术指标决定买卖时机，例如均线、MACD、趋势、情绪等。

当前策略没有一个明确的大盘择时模块。它的买卖时机主要来自：

| 当前机制 | 是否属于择时 | 说明 |
| --- | --- | --- |
| 每周第 2 个交易日调仓 | 弱择时 | 固定时间再平衡，不判断市场方向 |
| 30 日涨跌幅过滤 | 弱择时/风控 | 排除极端涨跌股票 |
| 涨停打开卖出 | 交易规则 | 更像 A 股涨停制度下的处理 |
| 止损模块 | 风控择时 | 默认关闭 |

这意味着：如果市场整体进入系统性下跌，当前策略默认仍会继续选股和持仓。

### 5.2 可以先加“市场状态观察”，不要急着加交易规则

作为新手，建议先观察，不要马上把择时信号写进交易逻辑。

可以先在回测后统计：

| 市场状态 | 观察策略表现 |
| --- | --- |
| 指数在 20 日均线上方 | 策略收益、回撤、胜率 |
| 指数在 60 日均线上方 | 策略收益、回撤、胜率 |
| 指数在 250 日均线上方 | 牛熊区间下策略是否稳定 |
| MACD 多头/空头 | 是否能过滤极端回撤 |

如果你发现某个市场状态下策略明显失效，再考虑把它变成交易规则。

### 5.3 择时优化实验

建议从简单、可解释的规则开始：

| 实验 | 规则 | 观察重点 |
| --- | --- | --- |
| 不加择时 | 当前基准 | 收益来源基线 |
| 沪深 300 收盘价高于 250 日均线才开仓 | 牛熊过滤 | 最大回撤是否下降，是否错过收益 |
| 中证 1000 高于 60 日均线才调仓 | 小盘环境过滤 | 小市值策略是否依赖小盘趋势 |
| 指数 20 日跌幅小于阈值才买入 | 极端风险过滤 | 急跌行情是否少亏 |
| 只降低仓位不清仓 | 温和择时 | 避免择时信号过度敏感 |

择时最容易过拟合。每加一个择时规则，都要同时看收益是否下降、交易次数是否变化、样本外是否有效。

## 6. 调仓：把 ch06 组合优化映射到代码

### 6.1 当前调仓方式

当前策略是最简单的组合构建方式：固定持仓数量，资金大致等额分配。

| 项目 | 当前值 |
| --- | --- |
| 调仓频率 | 周频 |
| 调仓日 | 每周第 2 个交易日 |
| 目标持仓数 | `g.stock_num = 10` |
| 买入方式 | 用可用资金平均买入待买股票 |
| 最小交易单位 | `LOT_SIZE = 100` |

调仓流程：

```text
每周第 2 个交易日
-> 重新计算目标股票列表
-> 卖出不在目标池且昨日未涨停的持仓
-> 用可用资金买入目标池中未持有股票
```

### 6.2 当前等权不是缺点，而是基准

ch06 里讲到等权重、市值加权、最小方差、最大分散度、风险平价、均值方差等方法。

当前策略接近“等权重组合”。等权很简单，但在量化研究里经常是很强的基准。不要一开始就急着换成复杂的组合优化。

建议优化顺序：

| 顺序 | 调仓优化 | 原因 |
| --- | --- | --- |
| 1 | 持仓数 5、10、20、30 | 最直接影响集中度和回撤 |
| 2 | 调仓日第 1/2/3/5 个交易日 | 检查是否有调仓日偶然性 |
| 3 | 周频 vs 双周频 vs 月频 | 检查换手和交易成本 |
| 4 | 等权 vs 市值反向加权 | 小市值策略中更敏感 |
| 5 | 等权 vs 风险平价 | 观察是否降低组合波动 |
| 6 | 最小方差/最大夏普 | 放到最后，因为更容易过拟合 |

### 6.3 组合风险要看“持仓数”和“权重”

当前默认 10 只持仓，单票理论权重大约 10%。如果某只股票连续跌停，对组合影响会很明显。

建议重点观察：

| 指标 | 为什么重要 |
| --- | --- |
| 平均持仓数 | 策略是否经常无法满仓 |
| 单票最大权重 | 个股风险是否过高 |
| 调仓换手率 | 是否被交易成本吃掉收益 |
| 行业集中度 | 是否押注少数行业 |
| 市值分布 | 是否过度集中在微盘 |
| 停牌/涨跌停持仓数量 | 是否有流动性风险 |

## 7. 风控：当前有止损，但默认关闭

当前止损参数：

| 参数 | 当前值 | 含义 |
| --- | --- | --- |
| `g.run_stoploss` | `False` | 止损总开关 |
| `g.stoploss_market` | `0.05` | 指数池平均跌幅超过 5% 时清仓 |
| `g.stoploss_limit` | `0.07` | 个股跌破成本 7% 时卖出 |
| `g.limit_days` | `5` | 卖出后短期禁止复购天数 |

新手常见误区：看到最大回撤高，就立刻加止损。止损确实可能降低回撤，但也可能切掉反弹收益。

建议做 4 组对照：

| 实验 | 目的 |
| --- | --- |
| 止损关闭 | 当前基准 |
| 只开个股止损 | 看单票风险是否下降 |
| 只开市场止损 | 看系统性风险是否下降 |
| 个股和市场止损都开 | 看组合效果是否过度卖出 |

## 8. 回测：用 ch07 的指标判断优化是否有效

ch07 提醒得很重要：只看年化收益是不可靠的。当前策略后续优化时，至少要固定输出下面这些指标。

| 类别 | 指标 | 用途 |
| --- | --- | --- |
| 收益 | 累计收益、年化收益、年度收益 | 看赚钱能力 |
| 风险 | 年化波动率、最大回撤、最长回撤修复期 | 看亏损体验 |
| 风险调整 | 夏普、Sortino、Calmar | 看承担风险后是否划算 |
| 相对收益 | Alpha、Beta、信息比率 | 看是否真的跑赢基准 |
| 交易 | 换手率、交易次数、交易成本占收益比例 | 看实盘可行性 |
| 持仓 | 平均持仓数、行业集中度、市值分布 | 看组合结构 |
| 稳健性 | 分年度、牛熊震荡分段、样本外 | 看是否过拟合 |

重点提醒：

```text
年化收益提高，但最大回撤更大、换手更高、Beta 更高，
不一定是策略变好了，可能只是风险暴露变大了。
```

## 9. Alpha/Beta 归因

这个策略尤其需要做 Alpha/Beta 归因，因为它主动筛选小市值股票。

建议至少用这些基准：

| 基准 | 用途 |
| --- | --- |
| 沪深 300 | 对比大盘蓝筹 |
| 中证 500 | 对比中盘风格 |
| 中证 1000 | 对比小盘风格 |
| 中证 2000 | 对比更小市值风格 |

简单理解：

| 结果 | 可能说明 |
| --- | --- |
| 对沪深 300 Alpha 很高，对中证 2000 Alpha 很低 | 收益可能主要来自小盘风格 |
| Beta 很高，回撤也高 | 策略更像高风险小盘组合 |
| Alpha 稳定，Beta 不极端 | 选股因子可能真的有贡献 |
| 牛市强、熊市弱 | 可能需要择时或风控 |

## 10. QMT 数据核验清单

在做任何参数优化前，先确认数据是对的。

| 数据项 | 核验方式 |
| --- | --- |
| 股票池 | 初始化日志里的股票池数量是否合理 |
| 日线行情 | `open/high/low/close/volume/amount` 是否有足够历史 |
| 涨跌停字段 | `upLimit/downLimit` 或候选字段是否能取到 |
| 停牌字段 | `suspendFlag` 是否有效 |
| 合约详情 | `InstrumentName/OpenDate/TotalVolume/FloatVolume` 是否能返回 |
| 市值单位 | 市值是否是亿元口径，不是元或万元误用 |
| 成交额单位 | `amount/money/turnover` 是否是元 |
| 本地板块名 | `INDEX_SECTOR_NAME_MAP` 是否匹配 QMT 客户端 |
| 账户资金 | `ContextInfo.capital` 是否和 QMT 回测设置一致 |
| 手续费滑点 | 脚本估算和 QMT UI 设置是否一致 |

如果数据字段错了，回测结果越漂亮越危险，因为你优化的是错误数据。

## 11. 推荐优化顺序

建议按“先可解释，后复杂”的顺序做。

### 第一轮：建立稳定基准

| 顺序 | 实验 | 观察重点 |
| --- | --- | --- |
| 1 | 固定当前参数，跑完整周期 | 得到基准指标 |
| 2 | 检查 QMT 数据字段 | 防止后面白优化 |
| 3 | 持仓数 5、10、20、30 | 集中度、回撤、换手 |
| 4 | 市值区间 5-30、5-50、10-80、20-100 亿 | 小市值暴露 |
| 5 | 流动性门槛 3000 万、5000 万、8000 万、1 亿 | 实盘容量 |

### 第二轮：验证因子贡献

| 实验 | 观察重点 |
| --- | --- |
| 只用低波动因子 | 低波动是否有效 |
| 只用低换手因子 | 低换手是否有效 |
| 去掉单个因子 | 是否存在拖累因子 |
| 调整波动率组/换手组权重 | 权重是否提升风险调整收益 |
| 分年度看因子表现 | 是否只在某几年有效 |

### 第三轮：加入择时和风控

| 实验 | 观察重点 |
| --- | --- |
| 止损开关对照 | 最大回撤和收益是否同时改善 |
| 指数均线过滤 | 是否降低熊市回撤 |
| 调仓频率变化 | 换手和交易成本 |
| 市场急跌过滤 | 极端行情保护效果 |

### 第四轮：做样本外验证

| 验证 | 目的 |
| --- | --- |
| 样本内/样本外切分 | 防止过拟合 |
| 牛市、熊市、震荡市分段 | 判断适用市场环境 |
| 手续费/滑点加倍 | 检查实盘脆弱性 |
| 换不同基准 | 判断 Alpha 是否稳定 |
| 排除最小市值股票 | 检查微盘依赖 |

## 12. 实验记录模板

每次优化建议都记录成下面这种格式：

| 项目 | 内容 |
| --- | --- |
| 实验名称 | 例如：持仓数从 10 改为 20 |
| 修改参数 | `g.stock_num = 20` |
| 理论依据 | ch06：分散持仓降低组合风险 |
| 预期结果 | 最大回撤下降，收益可能下降 |
| 实际结果 | 回测后填写 |
| 是否保留 | 是/否 |
| 原因 | 不只看收益，要结合风险和换手 |

不要一次改多个参数。一次改多个参数时，就不知道到底是哪一个改动产生了效果。

## 13. 当前最值得优先验证的问题

1. 收益是否主要来自 5 亿到 50 亿的小市值暴露。
2. 低波动因子和低换手因子哪个贡献更大。
3. 默认 10 只持仓是否太集中。
4. 5000 万成交额门槛是否足够覆盖你的预期资金规模。
5. 低换手因子和高成交额过滤是否存在冲突。
6. 止损关闭时，极端市场最大回撤是否可接受。
7. 加入简单指数择时后，是降低回撤，还是牺牲太多收益。
8. QMT 的股本、市值、成交额、涨跌停字段单位是否可靠。

## 14. 这份策略的学习路径

建议你按这个顺序学习和优化：

```text
先读懂选股链路
-> 确认每层过滤后的股票数量
-> 单独验证每个因子
-> 调整持仓数和市值区间
-> 再尝试择时/止损
-> 最后做样本外和成本压力测试
```

对应到 docs：

| 学习阶段 | 重点章节 | 当前策略任务 |
| --- | --- | --- |
| 选股入门 | ch04 多因子选股 | 看懂因子、标准化、权重和排序 |
| 择时入门 | ch05 均线/MACD | 先观察市场状态，再决定是否加入规则 |
| 调仓入门 | ch06 等权、风险平价、最小方差 | 先验证等权和持仓数，再考虑复杂权重 |
| 回测入门 | ch07 年化、回撤、Alpha/Beta、夏普 | 建立固定指标表，避免只看收益 |

## 15. 结论

当前版本适合作为“低波动小市值”策略的 QMT 回测基线。它的优势是逻辑清楚、因子可解释、参数集中；最大的风险是小市值暴露、流动性和数据字段可靠性。

后续优化的核心不是“把收益调高”，而是回答三个问题：

```text
选股因子是否真的有效？
调仓和仓位是否控制了风险？
回测结果是否能经得起样本外和交易成本检验？
```

只要你每次优化都能对应到 docs 中的一个知识点，并且保留完整实验记录，这个策略就会逐步从“能跑的代码”变成“你真正理解的量化策略”。
