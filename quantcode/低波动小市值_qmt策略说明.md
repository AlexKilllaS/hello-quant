# 低波动小市值 QMT 策略说明与优化指南

本文档对应策略文件：`quantcode/低波动小市值_qmt.py`。

这不是一份单纯的代码注释，而是一份学习型策略说明文档。它有两个目标：

1. 清晰描述当前 `低波动小市值_qmt.py` 到底在做什么。
2. 把你在 `docs` 里学到的量化选股、择时、调仓和回测知识，转化成后续优化这个策略时能直接使用的检查清单和实验路径。

重要边界：本文档解释当前策略如何工作，并给出优化实验方向。策略代码的量化逻辑仍以 `quantcode/低波动小市值_qmt.py` 为准。

## 0.1 2026-07-08 第一轮优化摘要

本轮优化吸收了 `quantcode/当前qmt策略分析优化.md` 的核心判断：当前 A 股小微盘不再适合无条件下沉暴露，策略需要先解决流动性、微盘风险、集中度和系统性回撤，再继续追求因子收益。

已落地到代码的改动：

| 优化方向 | 已落地规则 | 目的 |
| --- | --- | --- |
| 规避尾部微盘 | 总市值从 `5-50` 亿上移到 `10-80` 亿，并要求流通市值不低于 `8` 亿 | 降低退市、流动性和异常交易风险 |
| 提升可交易性 | 20 日日均成交额从 `5000` 万提高到 `8000` 万，并要求单日最低成交额不低于 `3000` 万 | 降低滑点和冲击成本 |
| 分散持仓 | 目标持仓从 `10` 只提高到 `20` 只，单票上限从 `15%` 降到 `8%` | 提高组合容错率 |
| 过滤极端走势 | 30 日动量区间从 `[-50%, +50%]` 收窄到 `[-30%, +30%]` | 排除短期暴涨暴跌标的 |
| 温和择时 | 使用 `399101.SZ` 前一交易日收盘价和 60/250 日均线决定目标仓位 | 趋势弱时降仓，不做激进清仓 |
| 避开开盘混乱 | 1 分钟周期下调仓检查从 `09:31` 后移到 `10:00` | 降低开盘价差和冲击成本 |
| 降低因子拥挤 | 因子不再 7 个等权，波动组约 60%，换手组约 40% | 减少同类因子重复投票 |

暂未直接写入代码的建议：财务质量因子、估值因子和行业中性化。原因是这些功能依赖 QMT 本地财务和行业字段，字段名、更新频率和历史可得性需要先在你的客户端确认；贸然接入会降低策略的可运行性。

## 0. 先建立全局地图

当前策略可以拆成 4 个学习模块：

| 学习模块 | docs 对应章节 | 当前策略中的位置 | 你要学会问的问题 |
| --- | --- | --- | --- |
| 量化选股 | `docs/ch04_量化选股策略` | `get_stock_list()`、`calc_barra_factors()`、`calc_composite_score()` | 为什么这些股票被选中，因子真的有用吗 |
| 量化择时 | `docs/ch05_量化择时策略` | `rebalance_check()`、`check_stop_loss()`、`trade_afternoon()` | 什么时候买卖，是否需要市场过滤 |
| 量化调仓 | `docs/ch06_量化调仓策略` | `buy_security()`、`order_target_value_()`、`g.stock_num` | 每只股票买多少，持仓是否太集中 |
| 量化回测 | `docs/ch07_量化回测` | QMT 回测结果和后续分析表 | 优化后是真的更好，还是只在某段行情碰巧更好 |

一句话概括当前策略：

```text
在中小盘股票池里，先做可交易性和风险过滤，
再用低波动、低换手相关因子排序，
每天检查目标池和小盘市场状态，只有触发缓冲池、权重漂移、目标仓位切换或月度校准条件时才调仓。
```

## 1. 策略结构

当前 QMT 策略入口：

| 函数 | 作用 |
| --- | --- |
| `init(ContextInfo)` | 初始化股票池、账户、参数和全局状态 |
| `handlebar(ContextInfo)` | 每根新 bar 检查一次 QMT 时间，并按任务时点触发策略流程 |

脚本仍保持 QMT 内置 Python 单文件形态，但结构上拆成初始化、QMT 适配、调度、组合、选股、交易几块。初始化里也显式区分 QMT 界面参数、策略参数、运行状态和账户状态。

推荐在 QMT 回测里使用 1 分钟周期。这样脚本可以用 bar 时间近似原聚宽的 `run_daily()` 调度：

| 触发时间 | 任务 | 作用 |
| --- | --- | --- |
| `09:25` | `prepare_stock_list()` | 同步持仓、记录昨日涨停持仓和禁买列表 |
| `10:00` | `rebalance_check()` | 每日做调仓检查；满足触发条件才卖出、买入、降仓或调权 |
| `10:30` | `check_stop_loss()` | 盘中止损检查，默认关闭 |
| `14:00` | `check_stop_loss()` | 第二次盘中止损检查，默认关闭 |
| `14:30` | `trade_afternoon()` | 处理涨停打开卖出和补仓 |

如果回测周期设置为日线，脚本会降级为“每天执行一次完整流程”。这种模式方便快速检查选股逻辑，但不能复刻原策略的日内时间点。这里没有使用 `ContextInfo.run_time()`，因为 QMT 回测中它不适合替代聚宽的 `run_daily()`。

## 2. 当前策略属于哪类策略

从 docs 的角度看，它是一个“多因子截面选股 + 小盘趋势仓位控制 + 缓冲触发调仓 + 简单风控”的策略。

它仍然不是纯择时策略。也就是说，它的核心收益来源不是预测大盘涨跌后频繁满仓/空仓，而是在股票池内部重新排序，选择当前更符合因子条件的股票；新增的择时模块只负责把目标股票仓位压到 100%、50% 或 30%。

当前策略主要暴露在这些风险上：

| 风险暴露 | 来源 | 为什么要关注 |
| --- | --- | --- |
| 小市值风险 | `10 亿 <= 总市值 <= 80 亿`，流通市值不低于 `8` 亿 | 仍有小市值 Beta，但已主动避开最尾部微盘 |
| 流动性风险 | 日均成交额和单日最低成交额双门槛 | 实盘滑点和冲击成本仍要压力测试 |
| 集中持仓风险 | 默认持有 20 只，单票上限 8% | 比 10 只更分散，但仍需防连续跌停和停牌 |
| 风格失效风险 | 偏低波动、低换手 | 强趋势牛市里可能跑输高弹性股票 |
| 数据字段风险 | 市值、涨跌停、停牌字段都依赖 QMT 本地数据 | 字段单位错了，排序就会错 |

### 2.3 副图诊断标识

脚本已经在 `handlebar(ContextInfo)` 末尾增加副图诊断输出。建议把 QMT 模型位置设为“副图”后回测，这样能在净值之外看到策略每根 bar 的状态变化。

| 标识 | 含义 | 主要用于判断 |
| --- | --- | --- |
| `D1基础过滤%` | 基础过滤后剩余股票数 / 初始股票池 | 股票池、ST、停牌、涨跌停、新股、高价股过滤是否过严 |
| `D2流动性%` | 流动性过滤后剩余股票数 / 初始股票池 | `amount` 字段是否正常，成交额门槛是否过高 |
| `D3市值池%` | 市值区间过滤后剩余股票数 / 初始股票池 | `TotalVolume/FloatVolume` 单位、市值区间是否合理 |
| `D4因子有效%` | Barra 因子有效股票数 / 候选股票数 | 历史行情、成交量、市值、基准收益是否足够计算因子 |
| `D5持仓完成%` | 当前持仓数 / 目标持仓数 | 是否长期没有买满，或者被涨跌停、停牌、资金约束卡住 |
| `D6现金比例%` | 当前可用资金 / 初始资金 | 是否长期闲置现金；这是资金利用观察值，不等同于组合现金占净值 |
| `D7新目标%` | 新进入 TopN 股票数 / 目标持仓数 | 排名变化和换仓压力是否过大 |
| `D8事件` | 交易和风控事件强度 | 红柱越高，代表调仓、买卖、止损、涨停打开或补仓事件越强 |
| `D9目标仓位%` | 小盘趋势仓位开关给出的目标股票仓位 | 当前是满仓、50% 控仓还是 30% 防守状态 |

`D8事件` 会配合文字标识：`检` 表示进入调仓检查，`持` 表示检查后继续持有，`调仓` 表示触发组合调整，`买/卖` 会显示当根 bar 的下单数量，`空池/历史不足/止损/市场止损/涨停开/补仓` 表示对应异常或风控事件。事件强度达到调仓或交易级别时，脚本还会画红色竖线，便于你把净值拐点和操作行为对齐。

事件强度不是收益预测信号，而是“这根 bar 策略做了什么”的强弱编码：

| `D8` 数值 | 图上标识 | 含义 | 应该怎么看 |
| --- | --- | --- | --- |
| `0` | 无文字 | 当根 bar 没有触发策略任务 | 忽略，不代表策略看空或看多 |
| `10` | `历史不足` | 日线历史不够，暂不调仓 | 扩大回测起始日期或补齐日线数据 |
| `30` | `检` 或 `持` | 进入调仓检查；若显示 `持`，表示检查后没有交易 | 这是正常巡检，不是买卖信号；红柱较低时通常只是持有状态 |
| `40` | `控仓` | 小盘趋势过滤给出的目标仓位低于满仓 | 看 `D9目标仓位%`，确认是 50% 中性防守还是 30% 风险防守 |
| `60` | `空池` | 选股结果为空 | 优先看 `D1-D4` 和日志，确认是过滤太严还是数据缺失 |
| `70` | `调仓` | 触发组合调整条件 | 看触发原因日志，是月度校准、持仓跌出缓冲池、新目标过多，还是权重漂移 |
| `80` | `补仓` | 涨停打开或止损卖出后，尝试把持仓补回目标数量 | 看 `D5` 是否恢复到 100，以及订单日志是否因为资金或手数不足跳过 |
| `90` | `涨停开` 或 `买n卖m` | 涨停打开卖出，或当根 bar 有买入 | 需要结合 K 线位置判断是否追高、卖飞或被涨停制度影响 |
| `95` | `买n卖m` | 当根 bar 有卖出；若同时有买入，会显示买卖数量 | 看换仓当天净值变化和后续收益，判断调仓质量 |
| `100` | `止损` 或 `市场止损` | 个股止损或市场止损 | 重点判断止损是否减少回撤，还是频繁卖在低点 |

同一根 bar 可能发生多个动作，脚本会保留强度最高的事件；如果当根 bar 有买卖，文字优先显示 `买n卖m`，例如 `买5卖0` 表示下了 5 笔买单、没有卖单，`买8卖3` 表示同时换入 8 笔、换出 3 笔。

截图里这种组合可以这样读：

| 看到的标识 | 说明 | 结论 |
| --- | --- | --- |
| `D1基础过滤%` 约 85 | 初始股票池经过上市、板块、ST、停牌、涨跌停、高价过滤后仍保留大部分股票 | 基础数据和基础过滤没有明显异常 |
| `D2流动性%` 约 80 | 成交额过滤后仍保留较多股票 | 8000 万日均和 3000 万单日最低成交额门槛不是当前主要瓶颈 |
| `D3市值池%` 约 25 | 市值区间过滤把股票池收缩到约四分之一 | 策略主要暴露在 10-80 亿中小市值区间，后续要重点做市值区间对照 |
| `D4因子有效%` 为 100 | 候选池股票都能算出完整 Barra 因子 | 当前光标处不是因子数据缺失问题 |
| `D5持仓完成%` 为 100 | 已达到目标持仓数量 | 没买满不是当前问题 |
| `D6现金比例%` 很低 | 可用现金接近满仓状态 | 现金闲置不是当前主要拖累 |
| `D7新目标%` 为 0 | 当前 TopN 没有新股票要求换入 | 排名稳定，调仓压力低 |
| `D8事件` 为 30 且文字为 `持` | 做了调仓检查，但没有触发交易 | 这是正常持有状态，不要把低红柱理解成卖出或风险信号 |
| `D9目标仓位%` 为 50 或 30 | 小盘趋势处在弱势区间 | 组合会按目标仓位降到半仓或三成仓，而不是继续无条件满仓 |

根据副图做排错时，可以按下面顺序看：

| 现象 | 优先排查 | 后续实验 |
| --- | --- | --- |
| `D1` 长期很低 | 本地板块成分、上市时间、ST、停牌、涨跌停字段 | 暂时只观察各过滤数量日志，不先放松策略条件 |
| `D2` 从 `D1` 大幅掉落 | `amount` 字段单位或 `g.min_avg_amount` 过高 | 流动性门槛 3000 万、5000 万、8000 万、1 亿分组回测 |
| `D3` 从 `D2` 大幅掉落 | 股本字段单位、市值区间太窄 | 市值区间 5-30、5-50、10-80、20-100 亿对照 |
| `D4` 长期偏低 | 日线历史不足、成交量或基准收益缺失 | 先补齐历史数据，再做单因子有效性测试 |
| `D5` 长期低于 100 | 可买标的不足、资金/手数约束、涨跌停限制 | 增加持仓数前，先检查订单日志和未买入原因 |
| `D6` 长期过高 | 资金没有部署出去或频繁卖出后未补齐 | 看 `D5` 和 `D8`，确认是空池、涨跌停、止损还是目标权重导致 |
| `D7` 经常很高 | 排名波动大，缓冲调仓仍有较高换手压力 | 提高 `buffer_multiplier`、`rebalance_min_new_targets` 或 `weight_drift_threshold` |
| `D8` 红柱密集且净值变差 | 策略可能过度交易或风控卖出过频繁 | 对照关闭止损、放宽调仓触发阈值、提高缓冲池倍数 |

## 3. 选股：把 ch04 多因子选股映射到代码

### 3.1 ch04 的学习框架

`docs/ch04_量化选股策略` 里，多因子选股大致是这条链路：

```text
确定目标和约束
-> 选择因子
-> 处理异常值
-> 标准化
-> 确定因子权重
-> 合成综合评分
-> 选出股票组合
-> 回测和调整
```

当前策略正好可以按这个框架拆开看。

| ch04 步骤 | 当前策略怎么做 | 对应代码 |
| --- | --- | --- |
| 目标和约束 | 只在指定股票池里选小市值、可交易股票 | `get_stock_list()` |
| 选择因子 | 使用低波动和低换手相关因子 | `calc_barra_factors()` |
| 异常值处理 | 对因子做 MAD 去极值 | `_mad_winsorize()` |
| 标准化 | 对因子做 Z-Score | `_zscore()` |
| 因子权重 | 波动组约 60%、换手组约 40% | `g.factor_weights` |
| 综合评分 | 加权得到 `composite_score` | `calc_composite_score()` |
| 股票筛选 | 得分越低越优，取前 N 只 | `rebalance_check()` |

### 3.2 股票池

当前股票池参数：

| 参数 | 当前值 | 含义 |
| --- | --- | --- |
| `g.factor_benchmark` | `000300.SH` | HSIGMA 使用的市场收益参考 |
| `g.report_benchmark` | QMT 回测参数里的基准 | 只用于初始化日志核对和回测报告语义区分 |
| `g.universe_index_code` | `399101.SZ` | 初始股票池，代码里固定映射为 QMT 本地板块名 `中小综指` |

代码通过 `INDEX_SECTOR_NAME_MAP` 把指数代码映射到 QMT 本地板块名。当前只使用 `中小综指`；如果该板块取不到成分股，脚本会直接抛异常终止，避免股票池被静默替换导致策略逻辑偏离预期。你在 QMT 里跑回测前，首先要看日志里的股票池数量和实际使用的板块名是否合理。

本版本对核心数据采用严格口径：板块名、行情字段、涨跌停计算口径、市值股本字段、Barra 因子依赖字段都不做多候选替换。缺字段时会打印字段名、时间、样本股票等排错信息，并在关键路径直接抛异常。

### 3.3 基础过滤

`get_stock_list()` 中的基础过滤顺序：

| 过滤器 | 当前逻辑 | 学习含义 |
| --- | --- | --- |
| `filter_listed_before` | 上市满约一年 | 避免新股数据不足或炒作期 |
| `filter_kcbj_stock` | 排除 4、8、688、30 开头股票 | 控制交易制度和风格差异 |
| `filter_st_stock` | 排除 ST、退市风险相关股票 | 基础风险过滤 |
| `filter_paused_stock` | 排除停牌或无有效量价数据股票 | 保证可交易 |
| `filter_limitup_stock` | 排除涨停且未持有股票 | 避免买不进去 |
| `filter_limitdown_stock` | 排除跌停且未持有股票 | 避免流动性风险 |
| `filter_highprice_stock` | 排除价格高于 `g.up_price` 的未持有股票 | 控制高价股和资金利用效率 |

新手优化时不要一开始就把这些过滤全部关掉。更好的方式是做“单项消融实验”：

```text
只关闭一个过滤器
-> 其他条件不变
-> 观察收益、回撤、换手、持仓数量变化
```

### 3.4 流动性过滤

当前参数：

| 参数 | 当前值 | 含义 |
| --- | --- | --- |
| `g.liquidity_days` | `20` | 成交额观察窗口 |
| `g.min_avg_amount` | `8e7` | 近 20 日平均成交额至少 8000 万元 |
| `g.min_daily_amount` | `3e7` | 近 20 日内每个有效交易日成交额至少 3000 万元 |

代码逻辑：

```text
近 20 日平均成交额 >= 8000 万元
且近 20 日最低单日成交额 >= 3000 万元
```

这一步很重要。小市值策略回测很容易好看，但实盘可能因为成交额太小、滑点太大而失真。

建议实验：

| 实验 | 观察重点 |
| --- | --- |
| 3000 万、5000 万、8000 万、1 亿 | 流动性门槛越高，收益是否下降，回撤是否改善 |
| 10 日、20 日、60 日窗口 | 短窗口更敏感，长窗口更稳定 |
| 分资金规模回测 | 100 万、500 万、1000 万资金下是否仍能成交 |

| `g.filter_threshold2` | `0.3` | 30 日涨幅上限 |

当前逻辑：

```text
-30% < 近 30 日涨跌幅 < +30%
```

这个过滤仍然不是强择时信号，而是风险过滤。相比原来的 ±50%，现在更主动排除短期暴涨暴跌的高风险标的。

建议实验：

| 参数组合 | 想验证什么 |

## 4. 因子：当前策略怎么打分

当前使用 7 个 Barra 量价类因子：

| 因子 | 类型 | 当前理解 | 排序方向 |
| --- | --- | --- | --- |
| `HSIGMA` | 波动率 | CAPM 回归残差波动率 | 越低越好 |
| `DASTD` | 波动率 | 指数加权日收益标准差 | 越低越好 |
| `CMRA` | 波动率 | 过去 12 个月累计收益区间 | 越低越好 |
| `STOM` | 换手 | 月度成交量/市值近似 | 越低越好 |
| `STOQ` | 换手 | 季度换手聚合 | 越低越好 |
| `STOA` | 换手 | 年度换手聚合 | 越低越好 |
| `ATVR` | 换手 | 指数加权换手率 | 越低越好 |

合成方式：

```text
原始因子
-> MAD 去极值
-> Z-Score 标准化
-> 按 g.factor_weights 加权
-> 得到 composite_score
-> composite_score 越低越优
```

当前权重：

```python
g.factor_weights = {
    "HSIGMA": 0.25,
    "DASTD": 0.20,
    "CMRA": 0.15,
    "STOM": 0.12,
    "STOQ": 0.10,
    "STOA": 0.10,
    "ATVR": 0.08,
}
```

这不再是 7 个因子等权，而是把 `HSIGMA/DASTD/CMRA` 这组波动类风险因子合计给到约 60%，把 `STOM/STOQ/STOA/ATVR` 这组换手和交易活跃度因子合计给到约 40%。这样做的目的不是追求复杂模型，而是降低同类换手因子重复投票造成的拥挤暴露。

### 4.1 因子优化不要直接上复杂模型

按照 ch04 的思路，优化因子前先做三件事：

1. 做单因子测试：每次只用一个因子排序，看它是否稳定贡献收益。
2. 做因子组测试：低波动组和低换手组分开测试。
3. 做因子相关性检查：如果两个因子高度相似，等权可能只是重复押注同一种风格。

建议实验：

| 实验 | 目的 |
| --- | --- |
| 只用 `HSIGMA/DASTD/CMRA` | 测试低波动因子族 |
| 只用 `STOM/STOQ/STOA/ATVR` | 测试低换手因子族 |
| 去掉单个因子 | 找出拖后腿因子 |
| 波动率组 70%、换手组 30% | 偏防御 |
| 波动率组 30%、换手组 70% | 偏低换手/低拥挤 |
| 等权 vs 手动权重 | 判断复杂权重是否真的优于简单等权 |

### 4.2 市值中性化是后续进阶点

ch04 里提到市值中性化和行业中性化。当前策略没有做中性化，而且还主动筛选了小市值股票，所以它很可能同时包含：

```text
小市值 Beta + 低波动/低换手 Alpha
```

你后续要重点验证：收益到底来自因子本身，还是主要来自小市值暴露。

可做实验：

| 实验 | 判断 |
| --- | --- |
| 和中证 1000/中证 2000 比较 | 小盘指数能否解释大部分收益 |
| 放宽市值到 20-100 亿 | 因子脱离微盘后是否仍有效 |
| 分市值组回测 | 小市值、中市值里因子表现是否一致 |
| 行业分布统计 | 是否集中在少数行业 |

## 5. 择时：当前策略有什么、缺什么

### 5.1 当前策略不是典型择时策略

ch05 讲的择时，重点是用客观技术指标决定买卖时机，例如均线、MACD、趋势、情绪等。

当前策略加入了一个很轻量的市场状态模块，但它只改变目标仓位，不直接替代选股逻辑。买卖时机主要来自：

| 当前机制 | 是否属于择时 | 说明 |
| --- | --- | --- |
| 每日 10:00 调仓检查 | 弱择时 | 固定时间检查组合状态，但交易由缓冲池、权重偏离和目标仓位切换触发 |
| 中小综指 60/250 日均线目标仓位 | 风控择时 | 趋势正常 100%，跌破 60 日线 50%，跌破 250 日线 30% |
| 30 日涨跌幅过滤 | 弱择时/风控 | 排除极端涨跌股票 |
| 涨停打开卖出 | 交易规则 | 更像 A 股涨停制度下的处理 |
| 止损模块 | 风控择时 | 默认关闭 |

这意味着：如果市场整体进入系统性下跌，当前策略不会立刻空仓，但会把新目标持仓金额压到较低比例，给组合留出现金缓冲。

### 5.2 当前小盘趋势仓位开关

代码里的 `_calc_market_timing_exposure()` 使用 `g.timing_index_code = "399101.SZ"`，并且只读取前一交易日以前的日线收盘价，避免使用当前交易日未来信息。

当前规则：

| 市场状态 | 目标仓位 |
| --- | --- |
| `399101.SZ` 收盘价高于 60 日线 | `100%` |
| 收盘价跌破 60 日线但仍高于 250 日线 | `50%` |
| 收盘价跌破 250 日线 | `30%` |

这个规则是防守型的：它不会预测短期涨跌，只是在小盘趋势变差时降低组合股票敞口。调仓时 `_target_position_values()` 会用 `组合总资产 * g.target_exposure` 作为目标股票资金，因此它会真实影响持仓金额。

### 5.3 择时优化实验

建议从简单、可解释的规则开始：

| 实验 | 规则 | 观察重点 |
| --- | --- | --- |
| 关闭择时 | `g.enable_market_timing = False` | 当前收益有多少来自仓位控制 |
| 沪深 300 收盘价高于 250 日均线才开仓 | 牛熊过滤 | 最大回撤是否下降，是否错过收益 |
| 中证 1000 或中证 2000 高于 60 日均线才满仓 | 小盘环境过滤 | 小市值策略是否依赖更宽的小盘趋势 |
| 指数 20 日跌幅小于阈值才买入 | 极端风险过滤 | 急跌行情是否少亏 |
| 目标仓位 100/70/40 | 更温和择时 | 是否比当前 100/50/30 少牺牲收益 |

择时最容易过拟合。每加一个择时规则，都要同时看收益是否下降、交易次数是否变化、样本外是否有效。

## 6. 调仓：把 ch06 组合优化映射到代码

### 6.1 当前调仓方式

当前策略已经从“固定交易日调仓”改为“风险预算缓冲调仓”。它每天在 `10:00` 做一次检查，但不等于每天交易。

| 项目 | 当前值 |
| --- | --- |
| 调仓检查频率 | 每个交易日 |
| 推荐执行时点 | 1 分钟周期下 `10:00` |
| 目标持仓数 | `g.stock_num = 20` |
| 缓冲池 | Top `g.stock_num * g.buffer_multiplier`，默认前 40 名 |
| 强制校准 | 每月首次检查触发一次 |
| 换仓触发 | 跌出缓冲池、TopN 新股票数量、目标仓位切换、权重漂移 |
| 权重方式 | 近 60 日波动率倒数加权，单票上限 8% |
| 最小交易单位 | `LOT_SIZE = 100` |

调仓流程：

```text
每个交易日
-> 09:25 准备持仓和昨日涨停列表
-> 10:00 检查日线历史是否足够
-> 计算小盘趋势目标仓位
-> 重新计算股票排序和 TopN 目标池
-> 计算反波动目标权重
-> 判断是否触发调仓
-> 触发后卖出不在目标池且昨日未涨停的持仓
-> 按目标权重补齐或降低持仓
```

触发条件：

| 条件 | 作用 |
| --- | --- |
| 当前空仓 | 建立初始组合 |
| 持仓数量不足 | 卖出后补齐组合 |
| 目标仓位和当前股票仓位偏离超过阈值 | 小盘趋势变弱或修复时降仓/加仓 |
| 月度强制校准 | 防止长期不动导致组合漂移 |
| 持仓跌出缓冲池达到阈值 | 避免质量明显变差还继续持有 |
| 新进入 TopN 的股票达到阈值 | 避免一两只股票排名噪声造成换手 |
| 已持有股票权重偏离超过 30% | 控制单票风险和组合漂移 |

### 6.2 为什么不用完整最小方差

ch06 里讲到等权重、市值加权、最小方差、最大分散度、风险平价、均值方差等方法。当前脚本采用的是“风险平价的简化版”：不用估计完整协方差矩阵，只用每只股票自己的近 60 日波动率做反向加权。

这样做的好处是更稳：输入参数少，QMT 回测里容易复现，也不容易因为协方差估计误差产生极端权重。单票上限 `g.max_position_weight = 0.08` 负责兜住集中度。

建议优化顺序：

| 顺序 | 调仓优化 | 原因 |
| --- | --- | --- |
| 1 | 持仓数 5、10、20、30 | 最直接影响集中度和回撤 |
| 2 | 缓冲池倍数 1.5、2、3 | 检查换手和收益是否平衡 |
| 3 | 新目标触发阈值 2、3、5 | 控制排序噪声带来的换手 |
| 4 | 权重漂移阈值 20%、30%、50% | 检查调权频率和单票风险 |
| 5 | 反波动窗口 40、60、120 日 | 观察风险预算是否稳定 |
| 6 | 单票上限 5%、8%、10%、15% | 控制集中度 |
| 7 | 完整最小方差/最大夏普 | 放到最后，因为更容易过拟合 |

### 6.3 组合风险要看“持仓数”和“权重”

当前默认 20 只持仓，反波动权重通常围绕 5% 上下浮动，单票上限为 8%。如果某只股票连续跌停，对组合仍有影响，但比原先 10 只、15% 上限的版本更能承受单票黑天鹅。

建议重点观察：

| 指标 | 为什么重要 |
| --- | --- |
| 平均持仓数 | 策略是否经常无法满仓 |
| 单票最大权重 | 个股风险是否过高 |
| 调仓换手率 | 是否被交易成本吃掉收益 |
| 行业集中度 | 是否押注少数行业 |
| 市值分布 | 是否过度集中在微盘 |
| 停牌/涨跌停持仓数量 | 是否有流动性风险 |

## 7. 风控：当前有止损，但默认关闭

当前止损参数：

| 参数 | 当前值 | 含义 |
| --- | --- | --- |
| `g.run_stoploss` | `False` | 止损总开关 |
| `g.stoploss_market` | `0.05` | 指数池平均跌幅超过 5% 时清仓 |
| `g.stoploss_limit` | `0.07` | 个股跌破成本 7% 时卖出 |
| `g.limit_days` | `5` | 卖出后短期禁止复购天数 |

新手常见误区：看到最大回撤高，就立刻加止损。止损确实可能降低回撤，但也可能切掉反弹收益。

建议做 4 组对照：

| 实验 | 目的 |
| --- | --- |
| 止损关闭 | 当前基准 |
| 只开个股止损 | 看单票风险是否下降 |
| 只开市场止损 | 看系统性风险是否下降 |
| 个股和市场止损都开 | 看组合效果是否过度卖出 |

## 8. 回测：用 ch07 的指标判断优化是否有效

ch07 提醒得很重要：只看年化收益是不可靠的。当前策略后续优化时，至少要固定输出下面这些指标。

| 类别 | 指标 | 用途 |
| --- | --- | --- |
| 收益 | 累计收益、年化收益、年度收益 | 看赚钱能力 |
| 风险 | 年化波动率、最大回撤、最长回撤修复期 | 看亏损体验 |
| 风险调整 | 夏普、Sortino、Calmar | 看承担风险后是否划算 |
| 相对收益 | Alpha、Beta、信息比率 | 看是否真的跑赢基准 |
| 交易 | 换手率、交易次数、交易成本占收益比例 | 看实盘可行性 |
| 持仓 | 平均持仓数、行业集中度、市值分布 | 看组合结构 |
| 稳健性 | 分年度、牛熊震荡分段、样本外 | 看是否过拟合 |

重点提醒：

```text
年化收益提高，但最大回撤更大、换手更高、Beta 更高，
不一定是策略变好了，可能只是风险暴露变大了。
```

## 9. Alpha/Beta 归因

这个策略尤其需要做 Alpha/Beta 归因，因为它主动筛选小市值股票。

建议至少用这些基准：

| 基准 | 用途 |
| --- | --- |
| 沪深 300 | 对比大盘蓝筹 |
| 中证 500 | 对比中盘风格 |
| 中证 1000 | 对比小盘风格 |
| 中证 2000 | 对比更小市值风格 |

简单理解：

| 结果 | 可能说明 |
| --- | --- |
| 对沪深 300 Alpha 很高，对中证 2000 Alpha 很低 | 收益可能主要来自小盘风格 |
| Beta 很高，回撤也高 | 策略更像高风险小盘组合 |
| Alpha 稳定，Beta 不极端 | 选股因子可能真的有贡献 |
| 牛市强、熊市弱 | 可能需要择时或风控 |

## 10. QMT 数据核验清单

在做任何参数优化前，先确认数据是对的。

| 数据项 | 核验方式 |
| --- | --- |
| 股票池 | 初始化日志里的股票池数量是否合理 |
| 日线行情 | `open/high/low/close/volume/amount/preClose/suspendFlag` 是否有足够历史 |
| 回测周期 | 推荐 1 分钟周期；日线周期只能做降级近似 |
| 复权口径 | 脚本固定 `front`，初始化日志会打印 QMT 界面复权设置用于核对 |
| 涨跌停价格 | 使用未复权 `preClose` 计算沪深主板普通股 10% 涨跌停价 |
| 停牌字段 | `suspendFlag` 是否有效 |
| 合约详情 | `InstrumentName/OpenDate/TotalVolume/FloatVolume` 是否能返回 |
| 市值单位 | `TotalVolume/FloatVolume` 是否为股数，市值按 `close * 股数 / 1e8` 得到亿元 |
| 成交额单位 | `amount` 是否是元 |
| 本地板块名 | `INDEX_SECTOR_NAME_MAP["399101.SZ"]` 是否匹配 QMT 客户端左侧板块 `中小综指` |
| 账户资金 | `ContextInfo.capital` 是否和 QMT 回测设置一致 |
| 手续费滑点 | 脚本估算和 QMT UI 设置是否一致 |
| 排错日志 | 初始化、行情字段缺失、过滤数量、市值缺失、Barra 跳过原因是否符合预期 |
| Barra换手分母 | 使用未复权 `close * FloatVolume` 生成流通市值序列，不再读取未确认的 K 线 `total_value` |

脚本会缓存 `get_instrumentdetail()` 的返回结果，减少同一轮回测中反复读取合约详情的开销。若仍然很慢，优先看耗时日志里最慢的阶段，而不是直接调参数。

如果数据字段错了，回测结果越漂亮越危险，因为你优化的是错误数据。本版本宁愿让回测尽早失败，也不自动换股票池、换字段或换排序逻辑。

## 11. 推荐优化顺序

建议按“先可解释，后复杂”的顺序做。

### 第一轮：建立稳定基准

| 顺序 | 实验 | 观察重点 |
| --- | --- | --- |
| 1 | 固定当前参数，跑完整周期 | 得到基准指标 |
| 2 | 检查 QMT 数据字段 | 防止后面白优化 |
| 3 | 持仓数 5、10、20、30 | 集中度、回撤、换手 |
| 4 | 市值区间 5-30、5-50、10-80、20-100 亿 | 小市值暴露 |
| 5 | 流动性门槛 3000 万、5000 万、8000 万、1 亿 | 实盘容量 |

### 第二轮：验证因子贡献

| 实验 | 观察重点 |
| --- | --- |
| 只用低波动因子 | 低波动是否有效 |
| 只用低换手因子 | 低换手是否有效 |
| 去掉单个因子 | 是否存在拖累因子 |
| 调整波动率组/换手组权重 | 权重是否提升风险调整收益 |
| 分年度看因子表现 | 是否只在某几年有效 |

### 第三轮：加入择时和风控

| 实验 | 观察重点 |
| --- | --- |
| 止损开关对照 | 最大回撤和收益是否同时改善 |
| 指数均线过滤 | 是否降低熊市回撤 |
| 缓冲池和触发阈值变化 | 换手和交易成本 |
| 市场急跌过滤 | 极端行情保护效果 |

### 第四轮：做样本外验证

| 验证 | 目的 |
| --- | --- |
| 样本内/样本外切分 | 防止过拟合 |
| 牛市、熊市、震荡市分段 | 判断适用市场环境 |
| 手续费/滑点加倍 | 检查实盘脆弱性 |
| 换不同基准 | 判断 Alpha 是否稳定 |
| 排除最小市值股票 | 检查微盘依赖 |

## 12. 实验记录模板

每次优化建议都记录成下面这种格式：

| 项目 | 内容 |
| --- | --- |
| 实验名称 | 例如：持仓数从 10 改为 20 |
| 修改参数 | `g.stock_num = 20` |
| 理论依据 | ch06：分散持仓降低组合风险 |
| 预期结果 | 最大回撤下降，收益可能下降 |
| 实际结果 | 回测后填写 |
| 是否保留 | 是/否 |
| 原因 | 不只看收益，要结合风险和换手 |

不要一次改多个参数。一次改多个参数时，就不知道到底是哪一个改动产生了效果。

## 13. 当前最值得优先验证的问题

1. 收益是否仍主要来自中小市值暴露，而不是低波动/低换手因子本身。
2. 低波动因子和低换手因子哪个贡献更大。
3. 默认 20 只持仓和 8% 单票上限是否足够分散。
4. 8000 万日均成交额和 3000 万单日最低成交额是否足够覆盖你的预期资金规模。
5. 低换手因子和高成交额过滤是否存在冲突。
6. 止损关闭时，极端市场最大回撤是否可接受。
7. 加入简单指数择时后，是降低回撤，还是牺牲太多收益。
8. QMT 的股本、市值、成交额、涨跌停字段单位是否可靠。

## 14. 这份策略的学习路径

建议你按这个顺序学习和优化：

```text
先读懂选股链路
-> 确认每层过滤后的股票数量
-> 单独验证每个因子
-> 调整持仓数和市值区间
-> 再尝试择时/止损
-> 最后做样本外和成本压力测试
```

对应到 docs：

| 学习阶段 | 重点章节 | 当前策略任务 |
| --- | --- | --- |
| 选股入门 | ch04 多因子选股 | 看懂因子、标准化、权重和排序 |
| 择时入门 | ch05 均线/MACD | 先观察市场状态，再决定是否加入规则 |
| 调仓入门 | ch06 等权、风险平价、最小方差 | 先验证缓冲调仓和反波动权重，再考虑复杂优化 |
| 回测入门 | ch07 年化、回撤、Alpha/Beta、夏普 | 建立固定指标表，避免只看收益 |

## 15. 结论

当前版本适合作为“低波动小市值”策略的 QMT 回测基线。它的优势是逻辑清楚、因子可解释、参数集中，并且对关键 QMT 数据接口采用严格口径；最大的风险是小市值暴露、流动性和数据字段可靠性。

后续优化的核心不是“把收益调高”，而是回答三个问题：

```text
选股因子是否真的有效？
调仓和仓位是否控制了风险？
回测结果是否能经得起样本外和交易成本检验？
```

只要你每次优化都能对应到 docs 中的一个知识点，并且保留完整实验记录，这个策略就会逐步从“能跑的代码”变成“你真正理解的量化策略”。
